What JSON Gets Wrong for Agents

Late Validation

Invalid tool calls detected after 50+ tokens. Wasted compute, wasted money.

Validate at token 3-5. Cancel immediately on hallucinated tools.

State Bloat

Every turn re-serializes full agent memory. O(n) growth kills long conversations.

Cryptographic patches. Send only what changed.

Repeated Keys

Arrays of objects repeat the same keys over and over. Massive waste.

Tabular mode. Headers once, rows stream efficiently.

Syntax Overhead

40% of tokens are quotes, colons, and commas. Pure noise.

Spaces and equals. Human-readable, machine-efficient.

Streaming Validation

The killer feature for tool-calling agents. Know if a tool call is valid before generation finishes.

# Streaming validation in action validator = StreamingValidator(tool_registry) for token in llm_stream: result = validator.push(token) if result.should_cancel: cancel_generation() # Stop bleeding tokens break

State Verification

Long-running agents accumulate state. GLYPH sends patches instead of full snapshots, with cryptographic verification.

Patch Operations

@patch base=a7f3c2... # SHA-256 of expected state memory[3]=Observation{...} # Set: replace value memory+Observation{...} # Add: append to list context.step~1 # Increment: atomic counter

The base= hash ensures patches apply to the expected state. Mismatch? Automatic resync. No silent corruption.

Tabular Mode

Agents often work with homogeneous data: search results, embeddings, database rows. JSON repeats keys for every item. GLYPH doesn't.

JSON (verbose)
[{"id": "doc_1", "score": 0.95}, {"id": "doc_2", "score": 0.89}, {"id": "doc_3", "score": 0.72}]
45 tokens
GLYPH Tabular
@tab [id score] doc_1 0.95 doc_2 0.89 doc_3 0.72 @end
18 tokens

Headers sent once. Rows stream efficiently. 60% smaller for typical datasets.

When to Use GLYPH

Use GLYPH When:

  • Building tool-calling agents
  • Agents have long-running state
  • Processing batch data in context
  • Token costs matter at scale
  • Multi-agent coordination needed
  • You want early validation

Maybe Not When:

  • Building REST APIs for browsers
  • Need maximum binary efficiency
  • Tooling requires strict JSON
  • Single-shot stateless requests

The GLYPH Difference

  • ✗ Commas → ✓ Spaces
  • ✗ Colons → ✓ Equals
  • ✗ "quoted" keys → ✓ bare keys
  • ✗ true/false → ✓ t/f
  • ✗ Resend all → ✓ @patch
  • ✗ Repeat keys → ✓ @tab tables
